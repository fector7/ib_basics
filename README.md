# Домашнее задание к занятию «Основы сетевой безопасности» - Лунев Федор Владимирович

## Ритейл

Вы являетесь экспертом ИБ в крупной ритейл компании. Компания в условиях пандемии COVID-19 переводит продажи в онлайн (интернет) и запускает Интернет-Сайт.

![](img/retail.png)

Какие средства защиты информации вы порекомендуете:

* \#1 - для разграничения доступа из Интернета к публикуемому сервису
* \#2 - для определения и подавления DDoS атаки на ваш интернет магазин
* \#3 - для определения и блокирования сложных WEB атак на ваш интернет-магазин, когда злоумышленники используют SQL инъекцию для получения доступа к вашей базе клиентов

---


* \#1 — Для разграничения доступа из Интернета к публикуемому сервису
> Решение: Межсетевой экран нового поколения (NGFW).

> Обоснование: Используется для организации DMZ (демилитаризованной зоны) и фильтрации трафика на сетевом и прикладном уровнях (L3-L7), блокируя несанкционированный доступ к веб-серверу из интернета.

* \#2 — Для определения и подавления DDoS атаки на ваш интернет магазин

> Решение: Специализированная система защиты от DDoS-атак (Anti-DDoS).

> Обоснование: Обеспечивает доступность сервиса, фильтруя паразитный трафик (как объемный, так и прикладной), предотвращая переполнение канала и исчерпание ресурсов сервера.

* \#3 — Для определения и блокирования сложных WEB атак (SQL-инъекции)

> Решение: Межсетевой экран уровня приложений (WAF — Web Application Firewall).

> Обоснование: Специализированное средство, понимающее логику работы веб-приложений. Оно блокирует атаки из OWASP Top 10 (включая SQL Injection и XSS), которые обычный межсетевой экран пропускает как легитимный HTTP-трафик.

## Промышленность

Вы являетесь экспертом ИБ в крупной промышленной компании. В компании появились новые инвестиции и руководство компании планирует модернизировать систему электронной почты и сделать более безопасным доступ сотрудников в Интернет.

![](img/industry.png)

Какие средства защиты информации вы порекомендуете:
* \#1 для разграничения доступа из Интернета в DMZ
* \#2 для защиты электронной почты от вирусов и спама
* \#3 - для определения и блокирования вредоносного ПО
* \#4 для контроля WEB трафика, URL фильтрации
* \#5 для разграничения доступа между корпоративной сетью и DMZ
* \#6 для защиты от вредоносного ПО на рабочей станции

---

* \#1 — Для разграничения доступа из Интернета в DMZ

> Решение: Внешний межсетевой экран (External Firewall / NGFW).

> Обоснование: Первый рубеж обороны. Фильтрует входящий трафик из интернета, скрывает топологию внутренней сети и защищает сервисы в DMZ от сканирования и прямых атак.

* \#2 — Для защиты электронной почты от вирусов и спама

> Решение: Шлюз безопасности электронной почты (Secure Email Gateway / ESG).

> Обоснование: Специализированный прокси для SMTP-трафика. Очищает поток писем от спама, фишинга и вредоносных вложений до их попадания на почтовый сервер, снижая нагрузку и риски.

* \#3 — Для определения и блокирования вредоносного ПО (в сети)

> Решение: Песочница (Sandbox) или Система предотвращения вторжений (IPS).

> Обоснование: Анализирует проходящий трафик и файлы. IPS блокирует сетевые эксплойты, а песочница эмулирует запуск файлов в изолированной среде для выявления неизвестных ранее угроз (Zero-day).

* \#4 — Для контроля WEB трафика, URL фильтрации

> Решение: Шлюз веб-безопасности / Прокси-сервер (Secure Web Gateway / Proxy).

> Обоснование: Контролирует доступ сотрудников в интернет. Позволяет блокировать опасные и непродуктивные категории сайтов (URL-фильтрация), а также проверяет скачиваемый контент на вирусы.

* \#5 — Для разграничения доступа между корпоративной сетью и DMZ

> Решение: Внутренний межсетевой экран (Internal Firewall).

> Обоснование: Критически важный элемент сегментации. Защищает внутреннюю сеть (LAN) от возможной компрометации сервисов в DMZ. Рекомендуется использовать оборудование другого вендора (отличного от #1) для повышения надежности.

* \#6 — Для защиты от вредоносного ПО на рабочей станции

> Решение: Хостовый антивирус (Endpoint Protection Platform / EPP).

> Обоснование: Последний рубеж защиты. Блокирует запуск вредоносного кода непосредственно на устройстве пользователя, если угроза проникла через сетевые фильтры (например, с USB-носителя).
